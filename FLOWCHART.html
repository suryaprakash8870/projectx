User Opens App

→

Enter 6-digit OTP (prototype: 123456). On success → ACTIVE status, placed in binary tree via BFS spillover.

2. Joining Request & Approval
2.1
Submit Joining Request
User pays ₹1,000 cash to admin offline, then clicks "Submit Joining Request" in app.
POST /api/joining/request
One request per user · Amount ₹1000 · Status: PENDING
→
2.2
Admin Reviews
Admin dashboard → Joining Requests tab. Sees all pending requests with member ID, name, mobile, referrer.
GET /api/joining/all
→
2.3
Approve → Payout Engine
Admin approves → triggers processJoiningPayout() which distributes ₹750 to 3 uplines and ₹1000 coupon to joiner.
POST /api/joining/:id/approve
3. Cycle-Based Payout Engine
3.1
Payout Resolution (₹1000 split)
Total ₹1000 joining fee splits into:
• ₹750 → 3 upline receivers (₹250 each, based on cycle table)
• ₹180 → GST record
• ₹70 → Company fee
₹250 × 3 Income ₹180 GST ₹1000 Coupon to Joiner
→
3.2
Dual-Path Resolution
Cycle 1 (joining cycle): Uses JOINER's upline path → pays L1, L2, L3
Cycles 2–5: Uses REFERRER's upline path → pays Self + 2 uplines
Missing upline → Company wallet gets the ₹250. After payout, referrer's cycle advances (2→3→4→5→2).
↓
3.3
Cycle Payout Table
CycleReceiversPath Used
1L1, L2, L3Joiner's
2Self, L4, L5Referrer's
3Self, L6, L7Referrer's
4Self, L8, L9Referrer's
5Self, L1, L2Referrer's
L1 = closest ancestor (direct parent) · L9 = farthest · Self = referrer earns ₹250

4.1
Coupon Wallet
₹1000 credited on joining approval. Can pay up to 50% of any product purchase.
COUPON
4.2
Purchase Wallet
Receives 2.5% cashback on every product purchase. Can transfer to Income Wallet.
PURCHASE
4.3

Receives ₹250 referral payouts. Withdrawable earnings. Also used for product purchases.
INCOME
4.4
GST Wallet

GST
5. Product Purchase Flow
5.1
Browse Products
Shop page shows product grid with categories. Filter by category.
GET /api/products
→
5.2
Place Order
Select product + qty. Payment: 50% Coupon + 50% Income wallet. 2.5% cashback → Purchase wallet.
POST /api/orders
Example: ₹500 product → ₹250 coupon + ₹250 income deducted · ₹12 cashback to purchase wallet
→
5.3
Revenue Split (10% fee)
Platform fee = 10% of order total. Split: 18% GST, 30% Company, rest to Users.
Example: ₹500 order → ₹50 platform fee → ₹9 GST + ₹15 Company + ₹26 Users

7. Binary Tree & Network
7.1
Binary Tree (UI Only)
Each user has LEFT and RIGHT children. Tree placement uses BFS spillover:
• Direct slot checked first (referrer's chosen leg)
• If occupied → BFS within that subtree, fill LEFT first, then RIGHT
• Tree structure does NOT affect earnings
512 hidden company users form the root chain · Real users start at IND000513+
7.2
9-Level Upline
Each user has up to 9 upline ancestors. Used for payout resolution (L1–L9).
GET /api/network/upline
